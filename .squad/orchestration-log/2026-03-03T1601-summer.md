## Outcome

Summer completed user-centric feature ideation with a decision document proposing 6 user-facing features grounded in EditLess's core promise: one place to see everything, plan, delegate, and review.

### Deliverable

**Decision File:** `.squad/decisions/inbox/summer-feature-ideation.md`

**Top 3 Recommendations for Immediate Implementation:**

1. **Live Session Dashboard Widget** (Medium effort, immediate ROI)
   - Per-agent status summary (active, waiting, idle, needs-review)
   - Color-coded + live updates via TerminalManager
   - Narrative: "Your team's heartbeat in one glance"

2. **Attention Queue** (Medium effort, scales to 10+ agents)
   - Unified inbox for agent input, reviews, and blockers
   - Quick actions: focus, skip, snooze
   - Narrative: "Never wonder which agent needs you"

3. **Session Notes & Decision Log** (Medium effort, enables team coordination)
   - Persist notes on sessions; visible to other agents (v0.3)
   - Foundation for knowledge transfer at scale
   - Narrative: "Your agents talk to each other through time"

**Implementation Roadmap:**
- v0.2.0: Dashboard Widget + Quick Agent Swap
- v0.2.1: Attention Queue
- v0.2.2: Session Notes & Decision Log
- v0.3+: Cross-Repo Linking, Smart Routing

**Design Principles:** One place to see everything, minimal context switching, async-first, tree-native UX, transparent (not magical).
