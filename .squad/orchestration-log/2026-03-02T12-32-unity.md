## Work Completed

Created `src/local-tasks-client.ts` — filesystem-based adapter for local markdown task files with YAML frontmatter.

### Deliverables

- **`LocalTask` interface** — Aligned with GitHub/ADO cross-platform abstraction
- **`fetchLocalTasks(folderPath: string): Promise<LocalTask[]>`** — Parses markdown files, extracts YAML frontmatter manually (no new dependencies)
- **`mapLocalState(task: LocalTask): UnifiedState`** — Maps todo/done status to open/active/closed using `session_id` as active-work signal
- **Graceful degradation** — Missing directory returns `[]`, malformed files skipped silently

### Technical Decisions

- Manual YAML parsing via `---` split + line-by-line key-value parsing — keeps dependency footprint flat
- State mapping to `UnifiedState` ensures consistency with GitHub/ADO cross-platform model
- Error boundary pattern matches existing `ado-client.ts` and `github-client.ts`
